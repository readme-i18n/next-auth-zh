### 配置

<Code>
  <Code.Next>

```ts filename="/auth.ts"
import NextAuth from "next-auth"
import Osso from "next-auth/providers/osso"

export const { handlers, auth, signIn, signOut } = NextAuth({
  providers: [Osso],
})
```

  </Code.Next>
  <Code.Qwik>
  
```ts filename="/src/routes/plugin@auth.ts"
import { QwikAuth$ } from "@auth/qwik"
import Osso from "@auth/qwik/providers/osso"

export const { onRequest, useSession, useSignIn, useSignOut } = QwikAuth$(
  () => ({
    providers: [Osso],
  })
)
```

  </Code.Qwik>
  <Code.Svelte>


### 注意事项

- 您可以在 Osso 管理界面（例如 https://yourInstance.com/admin/config）配置 OAuth 客户端，需要获取客户端 ID 和密钥，并设置允许的重定向 URI。
- SAML 单点登录与 OAuth 略有不同，对于每个希望通过 SAML 登录您应用的租户，您和客户都需在 Osso 管理界面和租户身份提供者的管理面板中完成多步骤配置。Osso 为 Okta 等提供商提供了文档，这些基于云的身份提供商也提供可用于测试的开发者账户。Osso 还提供了一个模拟身份提供商(Mock IDP)，无需注册身份提供商服务即可用于测试。
- `issuer` 应为完全限定域名，例如 `demo.ossoapp.com`
