# SailPoint ISC 提供商

SailPoint Identity Secure Cloud (ISC) 是一个面向身份与安全的企业级 SaaS 平台。要使用此 OAuth 集成，您需要拥有 ISC 租户。如果您是 SailPoint 客户或合作伙伴，请联系您的 SailPoint 客户经理获取详细信息。开发者可访问 [SailPoint 开发者社区](https://developer.sailpoint.com/discuss/)。

<Callout>
  该提供商未随任何 Auth.js
  软件包分发，因为这是一个企业级提供商，我们无法获取租户进行测试和兼容性验证。尽管如此，我们希望向用户提供此类提供商，因此我们将在文档页面（如本页）分享可复制粘贴的提供商配置。下方提供的提供商配置按原样提供，由拥有
  SailPoint 租户访问权限的社区成员提交。
</Callout>

### 创建 OAuth 客户端

首先，您需要在 SailPoint 管理控制台中创建一个客户端以获取 `clientId` 和 `clientSecret`。您可以按照此[指南](https://documentation.sailpoint.com/saas/help/common/api_keys.html?h=oauth+client#creating-an-api-key)操作，或遵循以下主要步骤：

1. 创建一个 OAuth 客户端 ()，授权类型选择：`AUTHORIZATION_TOKEN` 和 `REFRESH_TOKEN`。
2. 将重定向 URL 设置为与上述示例中的回调 URL 匹配。
3. 最后，选择作用域 `sp:scope:all`。
4. 点击"**创建**"并记录生成的 `clientId` 和 `clientSecret`。

### 环境变量

```sh
AUTH_SAILPOINT_ID=
AUTH_SAILPOINT_SECRET=
AUTH_SAILPOINT_BASE_URL=https://{tenant}.identitynow.com
AUTH_SAILPOINT_BASE_API_URL=https://{tenant}.api.identitynow.com
```

### 配置

与其他 Auth.js 提供程序不同，此配置无法从包中导入（有关详细信息，请参阅本页顶部的说明）。不过，您可以将以下对象复制粘贴到 `providers` 数组中以启用此提供程序。

<Code>
  <Code.Next>

```ts filename="/auth.ts"
import NextAuth from "next-auth"

export const { handlers, auth, signIn, signOut } = NextAuth({
  providers: [
    {
      id: "sailpoint",
      name: "SailPoint",
      type: "oauth",
      clientId: process.env.AUTH_SAILPOINT_ID!,
      clientSecret: process.env.AUTH_SAILPOINT_SECRET!,
      authorization: {
        url: `${process.env.AUTH_SAILPOINT_BASE_URL!}/oauth/authorize`,
        params: { scope: "sp:scopes:all" },
      },
      token: `${process.env.AUTH_SAILPOINT_BASE_API_URL!}/oauth/token`,
      userinfo: `${process.env.AUTH_SAILPOINT_BASE_API_URL!}/oauth/userinfo`,
      profile(profile) {
        return {
          id: profile.id,
          email: profile.email,
          name: profile.uid,
          image: null,
        }
      },
      style: { brandColor: "#011E69", logo: "sailpoint.svg" },
    },
  ],
})
```

### 用户资料

SailPoint 的 `userprofile` 端点会返回更多字段，但默认情况下 [用户表](/getting-started/database#models) 仅支持 `id`、`name`、`email` 和 `image`。因此，如果您想使用以下任何字段并且正在使用带有 Auth.js 的数据库适配器，请确保修改您所用适配器和数据库中的 `User` 表结构。然后您还可以从上述 `profile` 回调中返回以下任何字段。

SailPoint `userprofile` 端点响应中可用的字段包括以下内容。

```ts
type SailPointProfile = {
  tenant: string
  id: string
  uid: string
  email: string
  phone: string
  workPhone: string
  firstname: string
  lastname: string
  capabilities: string
  displayName: string
  name: string
}
```
