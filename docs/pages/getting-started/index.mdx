# 什么是 Auth.js？

Auth.js 是一个<abbr title="设计为可在不同 JavaScript 运行时环境中无缝工作，不依赖任何特定运行时的特性或行为">运行时无关</abbr>的库，基于标准 Web API 实现，深度集成多种现代 JavaScript 框架，提供开箱即用的认证体验，易于扩展且始终保障隐私安全！

本文档涵盖 `next-auth@5.0.0-beta` 及更高版本，以及 `@auth/*` 命名空间下的所有其他框架。`next-auth@4.x.y` 的文档仍可在 [next-auth.js.org](https://next-auth.js.org) 查阅。

选择您偏好的框架开始使用，或通过下方按钮查看示例应用部署及代码仓库。

## 认证方式

Auth.js 提供 4 种用户认证方式：

- [OAuth 认证](/getting-started/authentication/oauth)（_使用 Google、GitHub、LinkedIn 等账号登录_）
- [魔法链接](/getting-started/authentication/email)（_邮件服务提供商如 Forward Email、Resend、Sendgrid、Nodemailer 等_）
- [凭证认证](/getting-started/authentication/credentials)（_用户名密码、集成外部 API 等_）
- [WebAuthn](/getting-started/authentication/webauthn)（_通行密钥等_）
