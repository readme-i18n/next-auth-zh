# 内置页面

Auth.js 默认提供了一组页面，这些页面会在用户进行身份验证流程（注册、登录、登出、错误处理等）时展示。这一特性非常实用，让你初次使用该库时无需从零开始编写这些页面。生成的用户界面基于你在配置文件中指定的身份提供者。

如果没有传递 `providerId` 参数，`signIn` 函数会将用户重定向至登录页面。

import SignInPage from "../../../public/img/tutorials/sign-in-page.webp"

<Screenshot src={SignInPage} alt="Sign-in Page" />

本例中应用已配置了 [GitHub](/getting-started/providers/github) 和 [Credentials](/getting-started/providers/credentials) 提供者。

如果我们在 Auth.js 配置文件（`./auth.ts`）中添加 [Google](/getting-started/providers/google) 提供者，那么登录页面将会出现第三个使用 Google 登录的选项。

如需构建自定义登录页面，请参阅我们关于自定义登录页面的指南。
